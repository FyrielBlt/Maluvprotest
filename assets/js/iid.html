Google Maps API - vue js appelles les routes api/collier api/suivi retour json - SignalR pour voir localisation en direct :bibliothéque communication en temps réel ASP.net L’un des principaux problèmes rencontrés lors du développement de notre application a été la mise en place du suivi en temps réel des colis. Nous devions permettre aux clients de visualiser instantanément la position de leur colis sur une carte, sans avoir à actualiser la page. Les requêtes HTTP classiques (via Axios) ne convenaient pas, car elles nécessitent des appels réguliers au serveur, ce qui entraîne des retards et une surcharge. De plus, il fallait que le livreur transmette sa position GPS en continu, et que celle-ci soit diffusée immédiatement au client. La difficulté résidait donc dans le choix d’une technologie capable de gérer cette communication bidirectionnelle en direct. Après avoir étudié plusieurs options, nous avons identifié SignalR, proposé par Microsoft, comme une solution adaptée pour gérer efficacement les échanges en temps réel entre le serveur ASP.NET et l’interface Vue.js. --- analyse fonctionnelle : ce que le systéme doit faire (uml merise) analyse non fonctionnelle : comment le systéme doit étre ( sécurité perfermances ect .. ) ---- poo : code autour des objets ( données(attributs) + comportement (méthodes)) - encapsulation : regrouper les données et les méthodes sous une meme entité et protéger les données en controlant l'accées via des setters et getters - absraction : que les fonctionnalités nécessaires cacher les details téchnique voiture.demarrer() - héritage : une classe peut hériter des propriétés et des méthodes d'une architecturale - polymorhisme : une meme méthode peut avoir plusieurs comportement selon le type avantages ( réutilisabilités , extensiblité , proximité avec le monde réel )
